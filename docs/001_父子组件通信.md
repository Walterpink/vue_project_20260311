# 父子组件通信

在 Vue 3 中，**Props（属性）** 和 **Emit（事件）** 是组件通信的核心机制。随着 Composition API 和 `<script setup>` 的普及，它们的写法相比 Vue 2 更加灵活、类型安全且简洁。

下面从 **多种写法 + 用法场景 + TypeScript 支持** 三个维度，系统梳理 Vue 3 中 Props 与 Emit 的所有主流用法。

---

## 一、Props（父 → 子）

### ✅ 核心原则

- 父组件通过 **HTML 属性** 向子组件传递数据。
- 子组件通过 `defineProps` 声明接收。
- 所有 props **只读**，不可在子组件中直接修改。

---

### 📌 写法 1：TypeScript 泛型声明（推荐！）

```ts
// Child.vue
<script setup lang="ts">
const props = defineProps<{
  title: string          // 必填
  count?: number         // 可选
  isActive: boolean      // 必填
}>()

// 使用
console.log(props.title)
</script>
```

> ✅ 优点：类型推导精准，IDE 智能提示强，无运行时开销（纯编译时类型）。

---

### 📌 写法 2：带默认值（使用 `withDefaults`）

```ts
<script setup lang="ts">
interface Props {
  msg: string
  max?: number
  tags?: string[]
}

const props = withDefaults(defineProps<Props>(), {
  max: 100,
  tags: () => ['default'] // 注意：对象/数组必须用函数返回
})
</script>
```

> ⚠️ 为什么用函数？避免多个组件实例共享同一个引用。

---

### 📌 写法 3：运行时声明（兼容 JS 或复杂验证）

```ts
<script setup lang="ts">
const props = defineProps({
  title: {
    type: String,
    required: true
  },
  age: {
    type: Number,
    default: 0,
    validator: (value: number) => value >= 0
  }
})
</script>
```

> ✅ 适用场景：
>
> - 需要自定义验证（`validator`）
> - 项目未启用 TS
> - 需要动态默认值（但 `withDefaults` 已覆盖大部分场景）

---

### 📌 写法 4：Options API（传统方式，不推荐新项目）

```ts
<script lang="ts">
import { defineComponent } from 'vue'

export default defineComponent({
  props: {
    title: { type: String, required: true },
    count: { type: Number, default: 0 }
  }
})
</script>
```

> 💡 仅用于维护旧代码或混合架构。

---

### 🔁 父组件调用示例（统一）

```vue
<template>
  <Child 
    title="Hello" 
    :count="5" 
    :is-active="true"
  />
</template>
```

> 注意：
>
> - 静态值：`title="Hello"`（字符串）
> - 动态值：`:count="5"`（JS 表达式）
> - 布尔值简写：`<Child is-active />` 等价于 `:is-active="true"`

## 二、Emit（子 → 父）

### 核心原则

- 子组件通过 `$emit` 触发自定义事件。
- 父组件用 `@event-name` 监听。
- 推荐使用 **类型安全的 `defineEmits`**。

---

### 📌 写法 1：TypeScript 字面量类型（推荐！）

```ts
<script setup lang="ts">
// 声明多个事件及其参数类型
const emit = defineEmits<{
  (e: 'submit', data: { id: number; name: string }): void
  (e: 'close'): void
  (e: 'update:modelValue', value: string): void
}>()

// 使用
emit('submit', { id: 1, name: 'Alice' })
emit('close')
```

> ✅ 优势：完全类型安全，支持重载，IDE 自动补全。

---

### 📌 写法 2：字符串数组（简单场景）

```ts
<script setup lang="ts">
// 仅声明事件名，不校验参数类型
const emit = defineEmits(['submit', 'close'])

emit('submit', { /* 任意参数 */ }) // TS 不会报错
</script>
```

> ⚠️ 缺点：失去参数类型检查，不推荐在 TS 项目中使用。

---

### 📌 写法 3：运行时对象声明（带验证，较少用）

```ts
<script setup lang="ts">
const emit = defineEmits({
  submit: (data: unknown) => {
    if (!data || typeof data !== 'object') {
      console.warn('submit event expects an object')
      return false // 阻止 emit
    }
    return true
  }
})
</script>
```

> 💡 适用：需要运行时校验事件参数（一般用 TS 类型就够了）。

---

### 📌 写法 4：Options API（传统）

```ts
<script lang="ts">
export default defineComponent({
  emits: ['submit', 'close'], // 或 { submit: null, close: null }
  methods: {
    handleClick() {
      this.$emit('submit', { id: 1 })
    }
  }
})
</script>
```

> ❌ 新项目避免使用。

---

### 🔁 父组件监听示例

```vue
<template>
  <Child 
    @submit="handleSubmit"
    @close="onClose"
    @update:model-value="handleInput"
  />
</template>

<script setup lang="ts">
const handleSubmit = (data: { id: number; name: string }) => {
  console.log(data)
}
const onClose = () => { /* ... */ }
const handleInput = (value: string) => { /* ... */ }
</script>
```

## 三、高级用法 & 最佳实践

### 1. **`v-model` 的实现原理**

Vue 3 中 `v-model` 默认绑定 `modelValue` prop + `update:modelValue` 事件：

```vue
<!-- 父组件 -->
<Input v-model="searchText" />

<!-- 等价于 -->
<Input 
  :model-value="searchText" 
  @update:model-value="searchText = $event" 
/>
```

```ts
// 子组件
const props = defineProps<{ modelValue: string }>()
const emit = defineEmits<{ (e: 'update:modelValue', v: string): void }>()

const update = (val: string) => emit('update:modelValue', val)
```

> ✅ 多个 `v-model`：`v-model:title` → `title` + `update:title`

---

### 2. **Props 解构陷阱（不要直接解构！）**

```ts
// ❌ 错误：失去响应性
const { title } = defineProps<{ title: string }>()

// ✅ 正确：保持响应式引用
const props = defineProps<{ title: string }>()
// 使用 props.title
```

> 如果要用解构，需配合 `toRefs`：
>
> ```ts
> import { toRefs } from 'vue'
> const { title } = toRefs(props) // title 是 Ref
> ```

---

### 3. **Emit 事件命名规范**

- 普通事件：`submit`, `delete`, `change`
- 更新事件：`update:xxx`（用于 `v-model`）
- 避免使用原生 DOM 事件名（如 `click`），除非你真的要冒泡

---

## 四、总结对比表

| 功能 | 推荐写法 | 备注 |
|------|--------|------|
| **Props（TS）** | `defineProps<{ ... }>()` | 编译时类型，零运行时开销 |
| **Props（带默认值）** | `withDefaults(defineProps<...>(), { ... })` | 对象/数组用函数 |
| **Emit（TS）** | `defineEmits<{ (e: 'xxx', arg): void }>()` | 完整类型安全 |
| **v-model 支持** | `modelValue` + `update:modelValue` | Vue 3 默认约定 |

---

## ✅ 最终建议

- **新项目一律使用 `<script setup> + TypeScript`**
- **Props 用泛型接口 + `withDefaults`**
- **Emit 用字面量类型声明事件签名**
- **永远不要在子组件修改 props，用 emit 通知父组件**

这样写出的组件：**类型安全、可维护、高性能、符合 Vue 3 设计哲学**。
